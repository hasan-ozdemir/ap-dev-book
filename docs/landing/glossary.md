# Terminology Glossary

Use this glossary to decode the terms that appear across the Prodyum technology playbook. Each entry links back to the official guidance we expect teams to follow.

## .NET Multi-platform App UI (MAUI)
A single project UI framework that targets Android, iOS, macOS, and Windows with shared XAML, MVVM, and platform integrations.citeturn12search0

## .NET 9 Long-Term Support (LTS)
The 2024 LTS release of .NET that receives three years of security, quality, and servicing updates—our baseline for server, desktop, and mobile workloads.citeturn11search0

## .NET MAUI Essentials
The consolidated cross-platform API set (connectivity, sensors, secure storage, geolocation, etc.) surfaced through Microsoft.Maui.Essentials for .NET MAUI projects.citeturn17search0

## Hot Restart
A Visual Studio workflow that deploys .NET MAUI iOS apps directly to connected devices without the full App Store provisioning cycle, enabling faster inner loops on Windows.citeturn6search8

## Azure App Service
Microsoft’s managed hosting platform for web and API workloads with built-in scaling, deployment slots, and integrated DevOps hooks we rely on for companion services.citeturn13search0

## Azure Notification Hubs
A push notification service that fans out messages to Apple, Google, and Windows platforms with tagging and personalization support for mobile clients.citeturn18search0

## Azure DevOps Boards
The work management hub that provides backlogs, boards, and analytics; the New Boards experience became the default in 2024 and underpins our planning ceremonies.citeturn14search0

## GitHub Actions
GitHub’s native CI/CD platform; reusable workflows, protected environments, and matrix builds power our automated .NET build, test, and deployment pipelines.citeturn19search0

## Representational State Transfer (REST)
An architectural style for stateless HTTP APIs that surface resources via uniform URIs, HTTP verbs, and media types—used by our MAUI connectivity recipes.citeturn20search0

## Model-View-ViewModel (MVVM)
A presentation pattern that separates UI, presentation state, and business logic via bindings and commands, aligning with .NET MAUI’s data-binding model.citeturn21search0
